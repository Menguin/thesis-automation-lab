# Thesis Automation Framework Documentation
This document outlines the configuration, command-line operations, and directory architecture for the comparative analysis of three browser automation frameworks: Selenium, Playwright, and Cypress. 
---
## 1. Project Configuration (`package.json`, `playwright.config.js` & `cypress.config.js`)
In this Node.js environment, the configuration is split between package management and framework-specific execution rules:
1. **Dependency Management (`package.json`)**: Defines the required versions of Playwright, Cypress, and Selenium WebDriver. Executing `npm install` automatically resolves and installs these specific framework builds to ensure environmental consistency.
2. **Execution Scripts (`package.json`)**: Establishes aliases for complex command-line executions, standardizing the initialization and execution processes across all three frameworks.
3. **Playwright Environment (`playwright.config.js`)**: Playwright utilizes a dedicated configuration file to define the `baseURL` (currently set to the practice e-commerce application, saucedemo.com). This prevents hardcoded URLs in the test scripts and mirrors the configuration architecture of the Cypress station.
4. **Cypress Environment (`cypress.config.js`)**: Cypress utilizes a dedicated configuration file to define the `baseUrl` (currently set to the practice e-commerce application, saucedemo.com) and restrict test discovery strictly to the `cypress-station` directory. This prevents hardcoded URLs in the test scripts.

### Note on Environment Evolution: From `install` to `ci`
Initially, this project utilized `npm install` to generate the `package-lock.json` manifest. Once the environment was stabilized and the lock file was created, the configuration in `package.json` was upgraded to `npm ci` (Clean Install). This ensures that every subsequent execution uses the exact dependency versions recorded during the initial setup, providing the scientific "version-locking" required for a comparative thesis.

## 2. Command Line Interface (CLI) Reference
To execute the test suites or initialize the environment, open the integrated terminal and input the corresponding commands defined below:
| Command | Operational Description |
| :--- | :--- |
| `npm run setup-lab` | Initializes the project workspace. Installs all declared dependencies and browser binaries. (Required only upon initial setup). |
| `npm run run-playwright` | Executes the Playwright test suite and outputs structured benchmark data to `results/playwright.json`. |
| `npm run run-cypress` | Executes the Cypress test suite in headless mode (background execution) to ensure accurate performance benchmarking. |
| `npm run run-cypress-open`| Launches the Cypress interactive GUI for test debugging and visual verification. |
| `npm run run-selenium` | Executes the raw Node.js test scripts utilizing the Selenium WebDriver protocol. Execution time is logged to the console. |

## 3. Directory Architecture
The repository is structured to isolate framework-specific dependencies while maintaining a controlled environment.
* `/shared-data/` - Contains universal constants and environmental variables.
* `/selenium-station/` - Houses all test artifacts utilizing the Selenium WebDriver protocol.
* `/playwright-station/` - Houses all test artifacts utilizing Playwright's direct WebSocket architecture.
* `/cypress-station/` - Houses all test artifacts for Cypress's in-browser execution model.
* `/results/` - Houses structured benchmark output files for thesis analysis.
* `playwright.config.js` - The central execution ruleset for the Playwright framework.
* `cypress.config.js` - The central execution ruleset for the Cypress framework.
* `.gitignore` - Defines which files and directories are excluded from version control.

---
## 4. Auto-Generated Artifacts
During execution and installation, the frameworks automatically generate the following directories and files. **These should not be manually modified:**
* `node_modules/` - The local repository containing the downloaded source code for all frameworks and dependencies.
* `test-results/` - Playwright's automated directory for storing diagnostic evidence (screenshots, traces) of failed test executions.
* `results/playwright.json` - Structured JSON output generated by Playwright's test reporter, containing execution times and pass/fail data for thesis analysis.
* `package-lock.json` - A strict version-control manifest that ensures future installations match the exact sub-dependency versions used during this initial thesis research.

---
## 5. Version Control & Reproducibility
This project is version-controlled via Git and publicly available on GitHub. The `.gitignore` configuration intentionally excludes auto-generated and environment-specific files while preserving all benchmark data files under `/results/`. This ensures the raw thesis evidence remains visible and independently verifiable by examiners and reviewers.

### To Reproduce This Lab
1. Clone this repository.
2. Run `npm run setup-lab` to restore the exact dependency versions via `npm ci`.
3. Execute any of the `run-` commands to generate fresh benchmark data.
